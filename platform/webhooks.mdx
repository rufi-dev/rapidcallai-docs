## Overview

Webhooks notify your systems when events happen in RapidCall — like when a call ends, a transfer occurs, or a recording is ready.

## Setting Up Webhooks

Configure webhooks at two levels:

1. **Workspace level** — Settings → Workspace → Webhook URL (fires for all agents)
2. **Agent level** — Agent settings → Webhook URL (fires only for this agent)

Agent-level webhooks take priority over workspace-level.

## Webhook Payload

When a call ends, RapidCall sends a `POST` request to your webhook URL:

```json
{
  "event": "call.ended",
  "callId": "call_abc123",
  "agentId": "agent_xyz",
  "agentName": "Reception Bot",
  "to": "+15551234567",
  "fromNumber": "+15559876543",
  "direction": "inbound",
  "durationSec": 145,
  "startedAt": 1705312200000,
  "endedAt": 1705312345000,
  "outcome": "completed",
  "transcript": [
    { "role": "agent", "text": "Thank you for calling..." },
    { "role": "user", "text": "I'd like to book..." }
  ],
  "costUsd": 0.23
}
```

## Events

| Event | When |
|-------|------|
| `call.ended` | Call completed (includes full transcript) |
| `call.transferred` | Call was transferred to a human |
| `call.recording_ready` | Recording uploaded and available |

## Testing Webhooks

Use the **Test Webhook** button in agent settings to send a sample payload to your URL and verify it's working.

## Retry Policy

If your webhook endpoint returns a non-2xx status code, RapidCall will:
- Retry up to 3 times
- With exponential backoff (5s, 30s, 2min)

## Security

Verify webhook authenticity by checking:
- The `X-RapidCall-Signature` header (HMAC-SHA256 of the payload)
- The source IP matches RapidCall's servers

<Tip>
Use [webhook.site](https://webhook.site) or [requestbin.com](https://requestbin.com) for quick testing before setting up your production endpoint.
</Tip>
