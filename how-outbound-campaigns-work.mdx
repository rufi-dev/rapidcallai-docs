---
title: "How Outbound Campaigns Work"
---

An outbound campaign is any operation where your AI agent calls a list of contacts systematically rather than one at a time. The platform does not have a built-in campaign manager with a “start campaign” button — instead, campaigns are driven by external workflows that call the RapidCall API to trigger calls. This is by design: it gives you complete control over timing, sequencing, retry logic, and data flow, and it means the platform integrates cleanly with whatever tools you already use.

The typical campaign architecture has four components:

**1. A contact list. **This is where your leads or contacts live. It can be a Google Sheet, an Excel file, a CRM export, a database table, or any structured data source that contains the contact’s name, phone number, and any dynamic variables your agent needs (email, account number, appointment time, lead type, notes). The contact list is the source of truth for who gets called and what information the agent has when the call connects.

**2. A workflow engine. **This is the orchestration layer that reads contacts from the list and triggers calls via the API. Common choices include [Make.com](http://Make.com), n8n (self-hosted or cloud), Zapier, or a custom script. The workflow engine controls when calls are made, how many are made simultaneously, what happens when a call is not answered, and how outcomes are written back to the contact list.

**3. The RapidCall API. **The workflow calls the API to initiate each outbound call, passing the agent ID, the originating phone number, the destination number, and any dynamic variables. The API returns a call ID that you can use to track the call’s progress and retrieve results.

**4. Outcome logging. **After each call, the platform fires a webhook with the call outcome, transcript, recording URL, sentiment, extracted data, and any other fields you have configured. The workflow engine catches this webhook and writes the results back to the contact list, CRM, or database — closing the loop.

This four-part architecture is the same whether you are calling 10 contacts or 10,000. The difference at scale is how you configure the timing, concurrency, and retry logic.
