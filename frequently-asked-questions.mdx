---
title: "Frequently Asked Questions"
---

### Can I use RapidCall without technical knowledge?

Yes.

The platform is designed to be configured through the dashboard in plain language. You do not need to write code, use an API, or understand the underlying infrastructure in order to build and launch an agent.

Technical integrations such as webhooks, API access, and CRM connections are available, but they are optional.

### How many agents can I create?

There is no fixed limit on the number of agents you can create within a workspace.

You can create separate agents for different use cases, campaigns, teams, or clients as needed.

### Can I run multiple agents at the same time?

Yes.

Each agent operates independently, so you can run multiple agents simultaneously across different campaigns, phone numbers, and workflows.

### Can I use the same agent for inbound and outbound calls?

Yes.

A single agent can be assigned to both inbound and outbound routing. In practice, however, many users create separate inbound and outbound agents because the opening lines, flow structure, and call objectives are usually different.

### How quickly can I get an agent live?

A straightforward agent such as an appointment setter, lead qualifier, or follow-up agent can often be built, tested, and ready for live use within about 30 minutes.

More complex agents with multiple functions, integrations, and edge-case handlers usually take longer and often require several rounds of testing before they are production-ready.

### What happens if a provider goes down during a call?

RapidCall supports fallback configuration for both language models and voice providers.

If a primary provider fails and fallbacks have been configured, the platform can switch to the fallback option automatically to help keep calls running.

Fallbacks should be set up in the **Security & Fallback** section of the agent editor.

### Can I transfer calls to a human?

Yes.

The agent can use a transfer function to route the caller to a human team member. Transfers can be triggered by any condition you define in the prompt or call logic, such as:

- a direct request from the contact,
- a failed verification,
- a specific objection,
- or any other handoff condition in the workflow.

### Does the platform support multiple languages?

Yes.

RapidCall supports 18\+ languages, including:

- English with multiple accents,
- Spanish,
- French,
- German,
- Portuguese,
- Dutch,
- Polish,
- Russian,
- Arabic,
- Chinese,
- Japanese,
- Turkish,
- and more.

For multilingual operations, separate agents per language usually perform better than a single agent trying to handle multiple languages at once.

### Can I bring my own phone number?

Yes.

You can connect an existing business number through SIP trunking, or you can buy a new number directly through the platform.

Bringing your own number typically requires:

- E.164 formatting,
- a termination URI,
- and, if needed, authentication credentials.

### Is there a limit on call duration?

You can set a maximum call duration at the agent level.

There is no fixed platform-wide hard limit intended for normal use, but longer calls increase cost because billing is usage-based. For most workflows, it is best to set a reasonable limit that matches the use case.

### Can I listen to call recordings?

Yes.

If recording is enabled for the agent, each call includes a playable recording in the **Call Detail Panel**, along with a download option.

Recordings can also be referenced through the recording data available in workflow or webhook outputs where configured.

### Can I export my data if I leave?

Yes.

You can export:

- call history,
- recordings,
- transcripts,
- agent configurations,
- and contact or campaign data.

Available export formats include **CSV**, **XLS**, **JSON**, and **ZIP** archives for recordings. See the **Cancellation, Deletion and Data Export** section for full details.
