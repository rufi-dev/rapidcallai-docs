## Overview

Nodes are the building blocks of a workflow. Each node type serves a specific purpose in the conversation flow.

## Begin (Start Node)

<Icon icon="play" color="#7c3aed" /> The entry point of every workflow. Each workflow needs exactly one start node.

**Settings:**
- **Label** — Display name (e.g., "Call starts", "Inbound lead")

The start node automatically fires when a call connects, then flows to the next connected node.

---

## Conversation (Speech Node)

<Icon icon="message-square" color="#2563eb" /> Makes the agent say a specific message.

**Settings:**
- **Agent says** — The exact text the agent will speak
- **Label** — Display name for the node

Use this for scripted messages like greetings, confirmations, or specific information delivery.

---

## Conversation (AI Response Node)

<Icon icon="zap" color="#f97316" /> Lets the LLM respond freely based on context and instructions.

**Settings:**
- **Context/Instructions** — Additional instructions for this step (appended to the system prompt)
- **Label** — Display name

This is the most flexible node — the agent uses its LLM to generate a natural response. Use the context field to guide what the agent should discuss at this step.

---

## Listen Node

<Icon icon="ear" color="#7c3aed" /> Pauses and waits for the caller to speak.

**Settings:**
- **Silence timeout** — How many seconds to wait before moving on (default: 10)
- **Label** — Display name

Use after asking a question to give the caller time to respond.

---

## Logic Split (Condition Node)

<Icon icon="git-branch" color="#d97706" /> Branches the conversation based on keywords or intent.

**Settings:**
- **Keywords** — Comma-separated list of trigger words
- **Label** — Display name

Connect multiple outgoing edges and label them (e.g., "Yes" / "No"). The agent routes to the matching branch based on what the caller says.

**Example:** Keywords: `appointment, book, schedule` → If the caller mentions any of these words, take the "Yes" path.

---

## Call Transfer Node

<Icon icon="phone" color="#dc2626" /> Transfers the call to a human agent or another phone number.

**Settings:**
- **Transfer to** — Phone number in E.164 format (e.g., `+15551234567`)
- **Say before transfer** — Message the agent speaks before transferring

The call is transferred via SIP, and the AI agent disconnects.

---

## Ending (End Call Node)

<Icon icon="phone-off" color="#6b7280" /> Ends the conversation and hangs up.

**Settings:**
- **Goodbye message** — Final message before hanging up

Every conversation path should eventually lead to an end node.

---

## Function (Tool Node)

<Icon icon="wrench" color="#059669" /> Calls an external function during the conversation.

**Settings:**
- **Tool name** — The function to call (must be defined in the agent's tool configuration)
- **Label** — Display name

The agent executes the tool, gets the result, and uses it in the next response.

**Common tools:**
- `check_availability` — Query a calendar API
- `book_appointment` — Create a booking
- `lookup_account` — Search a CRM

---

## Wait Node

<Icon icon="clock" color="#0891b2" /> Pauses the conversation for a specified duration.

**Settings:**
- **Wait seconds** — How long to pause

Use sparingly — for cases like waiting for an API response or giving the caller time to complete an action.

---

## Node Summary

| Node | Purpose | Input Handle | Output Handle |
|------|---------|:---:|:---:|
| Begin | Entry point | ❌ | ✅ |
| Conversation | Agent speaks | ✅ | ✅ |
| AI Response | LLM responds freely | ✅ | ✅ |
| Listen | Wait for input | ✅ | ✅ |
| Condition | Branch logic | ✅ | ✅ (multiple) |
| Transfer | Transfer call | ✅ | ✅ |
| Function | Call a tool | ✅ | ✅ |
| Wait | Pause | ✅ | ✅ |
| Ending | Hang up | ✅ | ❌ |
