## Edges

Edges are the connections between nodes. They define the flow of conversation from one step to the next.

### Creating an Edge

1. Hover over a node's **output handle** (right side — circular dot)
2. Click and drag to another node's **input handle** (left side)
3. Release to create the connection

### Edge Labels

Edges can have labels that describe the transition condition. This is especially useful for condition nodes with multiple outgoing paths.

To add a label:
1. Click on the **source node** (the node where the edge starts)
2. In the right panel, scroll to **Connections**
3. Type a label for each outgoing edge (e.g., "Yes", "No", "Hot lead")

Labels appear as floating pills on the edge in the canvas.

### Deleting an Edge

Click on an edge to select it, then press `Delete` or `Backspace`.

## Condition Branching

The **Condition** node is the primary way to create branching logic.

### How It Works

```
                    ┌─── "Yes" ──→ [Transfer to booking]
[Condition Node] ───┤
                    └─── "No" ───→ [AI Response: General Q&A]
```

1. The condition node receives the caller's last message
2. It checks against the configured **keywords**
3. If any keyword matches, it follows the labeled path
4. If no match, it follows the default/other path

### Example: Booking Detection

**Keywords:** `appointment, book, schedule, reserve`

| Caller says | Match? | Path |
|-------------|--------|------|
| "I'd like to book an appointment" | ✅ `book`, `appointment` | → "Yes" |
| "What are your hours?" | ❌ No match | → "No" |
| "Can I schedule a visit?" | ✅ `schedule` | → "Yes" |

### Multiple Branches

You can create more than two branches from a condition node:

```
                    ┌─── "Billing" ──→ [Transfer to billing]
                    │
[What department?] ─┤─── "Sales" ───→ [Transfer to sales]
                    │
                    └─── "Other" ──→ [AI Response: Help desk]
```

Just add multiple outgoing edges with different labels and keyword sets.

## Best Practices

<CardGroup cols={2}>
  <Card title="Label every branch" icon="tag">
    Always label outgoing edges from condition nodes. Unlabeled edges are confusing to maintain.
  </Card>
  <Card title="Add a default path" icon="shield">
    Every condition should have a fallback path for unexpected responses.
  </Card>
  <Card title="Keep flows simple" icon="diagram-project">
    Avoid deeply nested conditions. If a workflow gets too complex, consider splitting into multiple agents.
  </Card>

    Use the Test Agent panel to verify every branch works as expected.
