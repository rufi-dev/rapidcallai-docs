## Base URL

```
https://your-server.com/api
```

For local development: `http://localhost:4000/api`

## Authentication

All API endpoints (except auth) require a Bearer token:

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://your-server.com/api/agents
```

Get a token by logging in via `/api/auth/login` or use an **API Key** from the dashboard.

## Response Format

All responses are JSON:

```json
{
  "data": { ... },
  "error": null
}
```

Error responses:

```json
{
  "error": "Unauthorized",
  "message": "Invalid or expired token"
}
```

## HTTP Status Codes

| Code | Meaning |
|------|---------|
| `200` | Success |
| `201` | Created |
| `400` | Bad request — invalid parameters |
| `401` | Unauthorized — missing or invalid token |
| `403` | Forbidden — insufficient permissions |
| `404` | Not found |
| `429` | Rate limited — too many requests |
| `500` | Server error |

## Rate Limits

| Endpoint Group | Limit |
|---------------|-------|
| Authentication | 5 requests / minute |
| General API | 100 requests / minute |
| AI generation (prompt assist) | 10 requests / minute |
| Billing operations | 1 request / minute |

Rate limit headers are included in responses:

```
X-RateLimit-Limit: 100
X-RateLimit-Remaining: 95
X-RateLimit-Reset: 1705312800
```

## Pagination

List endpoints support pagination:

```
GET /api/calls?limit=20&offset=0
```

| Parameter | Default | Max | Description |
|-----------|---------|-----|-------------|
| `limit` | 20 | 100 | Items per page |
| `offset` | 0 | — | Skip N items |
