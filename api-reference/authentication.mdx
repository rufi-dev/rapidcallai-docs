## Methods

RapidCall supports two authentication methods:

### 1. JWT Token (Session-based)

Login to get a JWT token:

```bash
curl -X POST https://your-server.com/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "password": "your-password"}'
```

Response:
```json
{
  "token": "eyJhbGciOiJIUzI1NiIs...",
  "user": { "id": "user_123", "email": "you@example.com" }
}
```

Use the token in subsequent requests:
```bash
curl -H "Authorization: Bearer eyJhbGciOiJIUzI1NiIs..." \
  https://your-server.com/api/agents
```

### 2. API Key

Create an API key from the dashboard (**Settings → API Keys**) and use it as a Bearer token:

```bash
curl -H "Authorization: Bearer rca_your_api_key_here" \
  https://your-server.com/api/agents
```

API keys are scoped to a workspace and never expire until revoked.

## Google OAuth

```bash
curl -X POST https://your-server.com/api/auth/google \
  -H "Content-Type: application/json" \
  -d '{"credential": "google-id-token"}'
```

## Register

```bash
curl -X POST https://your-server.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"email": "you@example.com", "password": "your-password", "name": "Your Name"}'
```

## Logout

```bash
curl -X POST https://your-server.com/api/auth/logout \
  -H "Authorization: Bearer YOUR_TOKEN"
```
