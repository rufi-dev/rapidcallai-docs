## Get Recording URL

```bash
curl -H "Authorization: Bearer YOUR_TOKEN" \
  https://api.rapidcall.ai/api/calls/call_abc123/recording-url
```

### Response

```json
{
  "url": "https://api.rapidcall.ai/api/calls/call_abc123/recording-playback?token=...",
  "expiresInSeconds": 1800
}
```

The URL is a signed playback URL that expires after 30 minutes.

## Playback URL

For browser playback with range support:

```bash
GET /api/calls/{id}/recording-playback?token=TOKEN&ws=WORKSPACE_ID
```

This endpoint streams the audio directly — use it as the `src` for an HTML `<audio>` element. Requires a valid signed token from the recording-url endpoint.

## Upload Recording

For manually uploading a recording (e.g., from an external source):

```bash
curl -X POST https://api.rapidcall.ai/api/calls/call_abc123/recording \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -F "file=@recording.ogg"
```

<Info>
Recordings are automatically captured via LiveKit Egress for all phone calls and web calls. Manual upload is only needed for special cases.
</Info>

## Recording Availability

After a call ends, the recording takes **30-40 seconds** to become available. This is normal — the recording engine needs to finalize and upload to S3.

Check `recording.status` on the call object: `"ready"` means the recording is available, `"pending"` means it is still processing.
