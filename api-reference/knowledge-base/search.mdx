---
title: Search Knowledge Base
api: POST /api/kb/search
description: "Semantic search across knowledge base documents"
---

## Request

```bash
curl -X POST https://your-server.com/api/kb/search \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What are your business hours?",
    "folder_id": "folder_abc123",
    "limit": 5
  }'
```

## Parameters

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `query` | string | ✅ | Search query (natural language) |
| `folder_id` | string | — | Limit to specific folder |
| `limit` | number | — | Max results (default: 5, max: 20) |

## Response

```json
{
  "results": [
    {
      "doc_id": "doc_abc123",
      "title": "Business Hours",
      "chunk": "We are open Monday through Friday, 9 AM to 5 PM EST. Saturday 10 AM to 2 PM. Closed Sundays and holidays.",
      "score": 0.92
    },
    {
      "doc_id": "doc_def456",
      "title": "Contact Information",
      "chunk": "Our office hours are 9-5 weekdays. For after-hours emergencies, call our on-call line at...",
      "score": 0.78
    }
  ]
}
```

## How It Works

1. Your query is converted to a vector embedding
2. The embedding is compared against all document chunks using cosine similarity (pgvector)
3. The most relevant chunks are returned, ranked by score

## During Calls

The voice agent automatically searches the knowledge base when it needs information. You don't need to call this endpoint manually for agent conversations — it happens automatically based on the agent's attached folders.

This endpoint is useful for:
- Testing what results your agent would see
- Building custom integrations
- Debugging knowledge base coverage
