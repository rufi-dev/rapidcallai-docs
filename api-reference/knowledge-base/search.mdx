---
title: Search Knowledge Base
api: POST /api/kb/search
description: "Search across knowledge base documents"
---

<ParamField header="Authorization" type="string" required>
  Bearer token or API key. Example: `Bearer rck_your_api_key`
</ParamField>

<ParamField body="query" type="string" required>
  Search query (max 500 characters).
</ParamField>

<ParamField body="folderIds" type="string[]" required>
  Folder IDs to search in (1-50 folders).
</ParamField>

<ParamField body="limit" type="number">
  Maximum number of results (default: 5, max: 10).
</ParamField>

## Request

```bash
curl -X POST https://api.rapidcall.ai/api/kb/search \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What are your business hours?",
    "folderIds": ["folder_abc123"],
    "limit": 5
  }'
```

## Response

```json
{
  "results": [
    {
      "id": "doc_abc123",
      "title": "Business Hours",
      "contentText": "We are open Monday through Friday, 9 AM to 5 PM EST...",
      "score": 0.92
    }
  ]
}
```

## How It Works

1. Your query is matched against document content using keyword and semantic search
2. The most relevant documents are returned, ranked by score
3. Results are scoped to the specified folders

## During Calls

The voice agent automatically searches the knowledge base when it needs information. You do not need to call this endpoint manually for agent conversations -- it happens automatically based on the agent's attached `knowledgeFolderIds`.

This endpoint is useful for:
- Testing what results your agent would see
- Building custom integrations
- Debugging knowledge base coverage
