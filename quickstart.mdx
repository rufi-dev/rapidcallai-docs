## Prerequisites

- [Docker](https://www.docker.com/) and Docker Compose installed
- An OpenAI API key (or any supported LLM provider key)

## 1. Clone and Start

```bash
git clone https://github.com/rufi-dev/rapidcallai.git
cd rapidcallai
cp .env.example .env
```

Edit `.env` and add your API keys:

```env
OPENAI_API_KEY=sk-your-openai-key
```

Start all services:

```bash
docker compose up -d
```

<Info>
First startup takes 2-3 minutes to pull images and initialize the database. Subsequent starts are much faster.
</Info>

## 2. Open the Dashboard

Navigate to [http://localhost:8080](http://localhost:8080) in your browser (or your configured domain if using Caddy).

1. **Register** — Create your account
2. **Create a workspace** — This isolates your agents, calls, and billing

## 3. Create Your First Agent

<Steps>
  <Step title="Go to Agents">
    Click **Agents** in the sidebar, then **Create Agent**
  </Step>
  <Step title="Configure the Agent">
    - **Name**: Give your agent a name (e.g., "Reception Bot")
    - **System Prompt**: Describe the agent's personality and instructions
    - **Voice**: Choose from available TTS voices
    - **LLM**: Select your preferred model (GPT-4o, Claude, Gemini, etc.)
  </Step>
  <Step title="Test with Web Call">
    Click the **Test** button to start a browser-based voice call with your agent. No phone number needed.
  </Step>
</Steps>

## 4. Connect a Phone Number (Optional)

To receive real phone calls:

1. Go to **Settings → Workspace** and add your Twilio credentials
2. Go to **Phone Numbers** and buy or import a number
3. Assign the number to your agent

Your agent is now live and answering phone calls!

## 5. Build a Workflow (Optional)

For more complex conversation flows:

1. Go to **Workflow Builder**
2. Start with a template (Receptionist, Sales, Support, or Appointment)
3. Customize nodes and connections
4. Assign the workflow to your agent
