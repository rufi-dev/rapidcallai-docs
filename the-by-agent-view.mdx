---
title: "The By Agent View"
---

The General view tells you how things are going overall. The By Agent view tells you why.

Switch to the By Agent tab and you get a table listing every agent in your workspace. Each row shows the agent’s name, total calls, successful calls, success rate, average duration, average latency, and average cost. This is where you stop looking at aggregate numbers and start comparing individual agents side by side. Just like looking at the performance of your human agents.

### **The Agent Table**

The table is sorted by call volume by default, so your busiest agents sit at the top. Each column tells you something different about that agent’s performance.

A few things jump out immediately when you scan a table like this. Success rate varies widely across agents — one might sit at 88% while another hovers at 46%. That gap is not random. It usually comes down to one of three things: the quality of the contact list each agent is working, the calling hours configured for that campaign, or the caller ID and number reputation. An agent with a 46% answer rate is not necessarily a bad agent — it might just be calling a cold list where half the numbers are stale.

Average duration tells you how conversations are going once they connect. An agent averaging 48 seconds is having short, pointed exchanges — that is typical for a website inbound handler or a quick qualification call. An agent averaging nearly 5 minutes is handling longer conversations, which could mean complex use cases (good) or an agent that rambles and gets stuck in loops (not good). Context matters.

Average latency is the technical pulse check. It shows the time between the contact finishing a sentence and the agent starting its response. Sub-1,000 ms feels natural. 1,200–1,400 ms is noticeable but acceptable for complex agents running heavier LLMs. Above 1,500 ms and contacts start to feel like something is off. If one agent’s latency is significantly higher than the others, check its voice stack configuration — the LLM and TTS combination is usually the driver.

### **Expanding an Agent**

Click the dropdown arrow on any agent row and the table expands to reveal that agent’s full performance breakdown — the same dashboard you see in the General view, but scoped to that single agent.

This expanded view shows five metric cards across the top: Total Calls (with completed count), Success Rate, Average Duration, Average Latency, and Average Cost per Call. Below that, the same daily volume bar chart, sentiment donut, and outcomes breakdown — all filtered to that one agent.

This is where you do real diagnostic work.

### **Comparing Agents**

The real power of the By Agent view is comparison. When two agents are running similar campaigns but one consistently outperforms the other, the difference is in the configuration — and the table helps you isolate it.

If Agent A has lower latency and higher sentiment than Agent B, and both run the same prompt, the voice stack is the variable. If they run the same stack but Agent A has longer average duration and more transfers, Agent A’s prompt is probably doing a better job of holding the conversation and reaching the handoff point.

Use the expanded views side by side (open one, scroll down, open the next) to compare sentiment distributions, outcome splits, and volume patterns. The answers are usually obvious once you see the data next to each other.

| 💡 Pro move: when testing a new prompt or voice stack, clone the agent, make the change on the clone, run both on the same campaign for a few days, and compare them in the By Agent view. That is a proper A/B test and it takes ten minutes to set up. |
