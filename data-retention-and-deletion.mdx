title: "Data Retention & Deletion"
---

### What data is stored

Every call processed through RapidCall generates operational data. Understanding what is stored helps you manage privacy, retention, and deletion obligations correctly.

| Data Type | Details |
| :-- | :-- |
| Call recordings | Audio files generated when recording is enabled for the agent |
| Transcripts | Text transcripts of the conversation generated for each call |
| Call metadata | Timestamps, duration, direction, phone numbers, call status, cost, and latency metrics |
| Post-call extracted data | Structured fields extracted from the transcript, such as outcome, sentiment, callback time, and custom fields |
| Agent configurations | Prompts, voice settings, function definitions, and knowledge base content stored per agent and per workspace |
| Account data | Registration details, account email, billing information, and team-member records |

### Your controls

RapidCall gives you direct control over the most sensitive categories of data.

### Disable recording

Call recording can be turned off at the agent level.

When recording is disabled, no audio file is generated or stored for that agent’s calls. Transcripts may still be generated where needed for call processing and platform functionality, but no recording is retained.

### Delete individual call records

You can delete individual call records directly from **Call History**.

Deleting a call record removes the associated stored call data for that record, including the recording, transcript, and extracted data linked to it.

### Export your data

You can export call history, recordings, transcripts, and agent configuration data before deleting records or closing a workspace.

Supported export formats include:

- **CSV**
- **XLS**
- **JSON**
- **ZIP** archives for recordings

See the **Cancellation, Deletion and Data Export** section for full export details.

### Close a workspace

If a workspace is closed, the associated data becomes inaccessible through the platform.

This includes:

- agents,
- call history,
- recordings,
- transcripts,
- and configuration data.

If you need to keep any of that information, export it before closure is processed.

### Responding to data subject requests

Under GDPR and similar privacy frameworks, individuals may have the right to request access to, correction of, or deletion of their personal data.

Because you are the data controller, those requests are your responsibility.

If a contact asks you to delete their data, the practical steps are usually:

- locate the relevant call record or records in **Call History**,
- delete the stored call data,
- remove the contact from any campaign or uploaded contact list,
- and delete the same data from any CRM or external systems where it was also stored.

If call data was pushed into your other systems through webhooks or integrations, you will need to remove it there as well.

If a contact sends a data request directly to RapidCall, they will be directed back to you as the data controller, since only you control the purpose, lawful basis, and wider context of the processing.
