## Quick Start

```bash
git clone https://github.com/rufi-dev/rapidcallai.git
cd rapidcallai
cp .env.example .env
# Edit .env with your API keys
docker compose up -d
```

Open the dashboard via your configured domain or Caddy reverse proxy.

## Services

Docker Compose starts these services:

| Service | Port | Description |
|---------|------|-------------|
| `api` | 8787 | Node.js API server |
| `dashboard` | 8080 (→ 80 internal) | React dashboard (Nginx) |
| `crm` | 8788 | CRM admin panel |
| `caddy` | 80, 443 | Reverse proxy with automatic TLS |
| `postgres` | 5432 | PostgreSQL database (with pgvector) |

## Docker Compose File

```yaml
services:
  api:
    build: ./server
    ports:
      - "8787:8787"
    env_file: .env
    depends_on:
      - postgres

  dashboard:
    build:
      context: ./client
      args:
        VITE_API_BASE: https://api.rapidcall.ai
    ports:
      - "8080:80"

  crm:
    build: ./crm-service
    ports:
      - "8788:8788"
    env_file: .env
    depends_on:
      - api

  caddy:
    image: caddy:2-alpine
    ports:
      - "80:80"
      - "443:443"
    volumes:
      - ./Caddyfile:/etc/caddy/Caddyfile:ro

  postgres:
    image: pgvector/pgvector:pg16
    environment:
      POSTGRES_DB: rapidcall
      POSTGRES_USER: rapidcall
      POSTGRES_PASSWORD: ${DB_PASSWORD}
    volumes:
      - pgdata:/var/lib/postgresql/data
    ports:
      - "5432:5432"

volumes:
  pgdata:
```

## Database Initialization

On first startup, the server automatically:
- Creates all required tables
- Sets up pgvector extension for knowledge base embeddings
- Runs any pending migrations

## Updating

```bash
git pull
docker compose build
docker compose up -d
```

## Health Check

Verify the server is running:

```bash
curl http://localhost:8787/health
# {"ok": true}
```

## Logs

```bash
# All services
docker compose logs -f

# Specific service
docker compose logs -f api
docker compose logs -f dashboard
```
