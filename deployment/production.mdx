## Security Checklist

<Warning>
Complete this checklist before going live with real phone calls.
</Warning>

- [ ] **Set strong `JWT_SECRET`** вҖ” At least 64 random characters
- [ ] **Set `AGENT_SECRET`** вҖ” Unique secret for agent-server communication
- [ ] **Enable HTTPS** вҖ” Use a reverse proxy (Nginx, Caddy) with TLS certificates
- [ ] **Restrict CORS** вҖ” Set `CORS_ORIGINS` to your domain only
- [ ] **Use environment variables** вҖ” Never hardcode secrets in code
- [ ] **Database backups** вҖ” Automated daily backups of PostgreSQL
- [ ] **Rate limiting** вҖ” Enabled by default, verify limits match your traffic
- [ ] **Firewall rules** вҖ” Only expose ports 80/443 publicly

## Reverse Proxy (Caddy)

RapidCall AI uses Caddy for automatic TLS and reverse proxying:

```
api.yourdomain.com {
    reverse_proxy localhost:8787
}

dashboard.yourdomain.com {
    reverse_proxy localhost:8080
}

crm.yourdomain.com {
    reverse_proxy localhost:8788
}
```

Caddy automatically provisions and renews TLS certificates via Let's Encrypt.

### Alternative: Nginx

```nginx
server {
    listen 443 ssl;
    server_name api.yourdomain.com;

    ssl_certificate /etc/ssl/certs/yourdomain.crt;
    ssl_certificate_key /etc/ssl/private/yourdomain.key;

    location / {
        proxy_pass http://localhost:8787;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

## Scaling

### Horizontal Scaling

- **Server** вҖ” Stateless, can run multiple instances behind a load balancer
- **Agent** вҖ” Each agent process handles one call at a time. Scale by adding more processes
- **Voice Engine** вҖ” Supports distributed deployment with multiple nodes

### Database

- Use **connection pooling** (PgBouncer) for high concurrency
- Enable **pgvector** index for knowledge base searches
- Regular `VACUUM ANALYZE` for performance

## Monitoring

### Health Endpoints

```bash
# Server health
curl https://api.yourdomain.com/health

# Metrics (Prometheus format)
curl https://api.yourdomain.com/metrics
```

### Key Metrics to Monitor

- Server response time (p50, p95, p99)
- Active calls count
- LLM latency per call
- Database connection pool utilization
- Agent process CPU/memory usage
- Call success/failure rate

## Backup Strategy

```bash
# Daily PostgreSQL backup
pg_dump -h localhost -U rapidcall rapidcall | gzip > backup_$(date +%Y%m%d).sql.gz

# Keep 30 days of backups
find /backups -name "*.sql.gz" -mtime +30 -delete
```

## Voice Engine Deployment

For production, deploy the voice engine on dedicated infrastructure with sufficient resources for real-time audio processing. Consider multi-region deployment for global coverage and low latency.

<Tip>
Deploy the voice engine in the same region as your Twilio SIP trunks for optimal audio quality and minimal latency.
</Tip>
