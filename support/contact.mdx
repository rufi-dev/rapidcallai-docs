If you have worked through the relevant troubleshooting steps and the issue is still not resolved, contact the RapidCall team directly.

## Support channels

- **Live chat** — available inside the dashboard (bottom-right corner)
- **Email** — hello@rapidcall.ai
- **Discord** — [discord.gg/rapidcallai](https://discord.gg/rapidcallai)

Live chat is usually the fastest option, and most platform issues can be diagnosed quickly when the right details are provided.

## What to include

For faster support, include the information relevant to the issue:

### For call issues

- call ID or recording URL from Call History,
- agent name,
- phone number involved,
- and a short description of what went wrong.

### For billing issues

- invoice date and amount,
- the specific line item in question,
- and the date range you want reviewed.

### For agent behaviour issues

- call ID,
- the point in the script where the issue happened,
- what you expected to happen,
- and what actually happened (include the transcript if possible).

### For API integration issues

- the endpoint you are calling,
- the full request and response (with sensitive fields redacted),
- and the error message or HTTP status code.

### For phone number issues

- the phone number in question,
- whether it is purchased through RapidCall or connected via SIP,
- and the error or unexpected behaviour you are seeing.

## Response times

- **Live chat**: typically within minutes during business hours
- **Email**: within 24 hours
- **Critical production issues**: mention "urgent" in the subject — these are prioritised

## Before contacting support

Check these common solutions first:

1. **Agent not answering calls** — make sure the agent is published and the phone number is assigned
2. **No transcript or recording** — check that the call lasted more than a few seconds and the agent connected successfully
3. **Billing discrepancy** — check the Usage tab in Billing for a per-call cost breakdown
4. **API returning 401** — verify your API key is correct and not expired
5. **Phone number not ringing** — confirm the SIP trunk or Twilio number is properly configured in Phone Numbers settings
