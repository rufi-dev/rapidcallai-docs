## Inbound Call Flow

<Steps>
  <Step title="Call arrives">
    A customer dials your phone number. The call is routed to the RapidCall platform.
  </Step>
  <Step title="Agent is identified">
    RapidCall looks up which agent is assigned to the phone number and loads the agent configuration (prompt, voice, LLM, tools, knowledge base).
  </Step>
  <Step title="Voice session starts">
    A real-time voice session is created and the phone call is bridged in.
  </Step>
  <Step title="Greeting">
    The agent speaks its configured greeting message (e.g., "Thank you for calling Acme Dental, how can I help you?").
  </Step>
  <Step title="Conversation loop">
    For each turn:
    1. **Speech recognition** transcribes the caller's speech in real-time
    2. **LLM** generates a response based on context and system prompt
    3. **Text-to-speech** converts the response to voice and plays it back
    4. **Tools** are executed if needed (booking, transfer, lookups)
  </Step>
  <Step title="Call ends">
    When either party hangs up:
    - Transcript is saved
    - Recording is uploaded to storage
    - Call metrics (duration, latency, cost) are recorded
    - Webhook fires if configured
  </Step>
</Steps>

## Outbound Call Flow

Outbound calls follow a similar flow, but RapidCall initiates the call:

1. **API or Dashboard** triggers the call with a phone number to dial
2. A voice session is created and the outbound call starts
3. When the callee answers, the voice agent joins and begins the conversation
4. Same conversation loop as inbound

## Web Call Flow

Web calls use WebRTC directly through the browser -- no phone number involved:

1. User clicks **Test** in the dashboard
2. Browser connects to a voice session via WebRTC
3. Voice agent joins the same session
4. Audio flows directly between browser and agent

<Info>
Web calls are free -- they do not use telephony minutes. Use them for testing and development.
</Info>

## Call Recording

Recordings are captured automatically:

- Audio is recorded during the voice session
- When the call ends, the recording is uploaded to storage
- Upload takes approximately 30-40 seconds after call end
- Recordings are accessible via the dashboard or API

## Call Transfer

During a conversation, the agent can transfer the call to a human:

1. The LLM decides to use the `transfer_call` tool
2. The agent says a transition message (e.g., "Let me connect you to a specialist")
3. A SIP transfer is initiated to the target phone number
4. The caller is connected to the human agent
5. The AI agent disconnects from the call

Transfer destinations can be:
- External phone numbers (E.164 format)
- SIP endpoints
