---
title: "Logic Issues"
---

Logic issues are when the agent connects and speaks, but does not follow the script correctly — it skips steps, gives wrong information, fails to trigger a function, or goes off-script entirely. These are almost always prompt problems, not platform problems.

**Agent skips steps in the script. **If the agent jumps from Step 1 to Step 4, the main script section of the prompt is not structured tightly enough. Use explicit numbered steps with clear “wait for response” gates between them. The agent should not advance until the contact has responded. If steps blend together, add explicit instructions: “Do NOT proceed to Step 3 until the contact has confirmed their date of birth.”

**Agent gives information it should not have. **If the agent is sharing details that are not in the prompt or knowledge base, it is hallucinating. Tighten the prompt constraints: “Only share information explicitly provided in this prompt or the knowledge base. If you do not have the answer, say so.” Adding a clear boundary instruction reduces hallucination significantly.

**Agent fails to call a function. **If the agent is supposed to book an appointment, transfer a call, or send an SMS but does not, check three things. First, is the function defined in the agent’s Functions tab? Second, does the prompt instruct the agent to call the function at the right moment? Third, is the function name in the prompt exactly matching the function name in the configuration? Mismatched names are the most common cause of function failures.

**Agent does not handle an objection correctly. **If the agent falls apart when the contact pushes back, you either do not have a handler for that scenario or the handler is too vague. Review the transcript to see exactly what the contact said, then write a specific handler for that response pattern. Generic handlers (“if the contact objects, be understanding”) produce generic behaviour. Specific handlers (“if the contact says they are too busy, ask whether mornings or afternoons work better”) produce specific, useful behaviour.

**Agent uses the wrong dynamic variable. **If the agent reads blank values, says “undefined,” or uses the wrong name, the dynamic variables in the prompt do not match the variables being passed by the campaign or workflow. Check that every {{variable}} in the prompt has a matching field in the contact data. 

**Agent worked yesterday but broke today. **If nothing changed on your side, check two things: did you save without publishing (meaning the agent reverted to the last published version), or did you make a change and publish it without testing? Use the History tab to roll back to a previous version if needed.

| 🔧 Debugging workflow: Open the call in Call History, read the full transcript, and find the exact moment the agent went wrong. Compare that moment to the corresponding section of the prompt. The gap between what the prompt says and what the agent did is where the fix is. |
