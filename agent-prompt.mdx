---
title: "Agent prompt"
---

The prompt is the single most important part of your agent. Written in plain language, it defines the agent’s identity, objective, how it handles common responses, how it handles edge cases, and when to end the call. Time spent here pays off directly in call quality and conversion rates.

A complete prompt has five sections. Role defines the agent’s persona, tone, and core objective. Context covers dynamic data the agent uses to personalise each call — contact name, callback time, product interest — passed in from your CRM or contact list. Availability sets business hours and booking rules if the agent schedules appointments. Guidelines define ongoing behaviour: staying on topic, handling objections, keeping responses tight, managing transfers. Main Script is the step-by-step conversation flow from introduction to close, with branching logic at each stage.

Optionally, you can add Handlers — named fallback responses for specific situations like the contact being busy, hostile, already dealt with elsewhere, or a booking action failing. Not mandatory, but highly effective for production agents.
