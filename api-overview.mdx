The RapidCall API lets you programmatically manage agents, trigger calls, retrieve call data, and integrate RapidCall into your own applications.

## Base URL

```
https://api.rapidcall.ai
```

## Authentication

All API requests require a Bearer token in the `Authorization` header:

```bash
curl https://api.rapidcall.ai/api/agents \
  -H "Authorization: Bearer YOUR_API_KEY"
```

You can generate an API key from **Dashboard > Settings > API Keys**.

## Response format

All responses are JSON. Successful responses return the data directly. Errors return:

```json
{
  "error": "Description of what went wrong"
}
```

## Rate limits

- **60 requests per minute** per API key for most endpoints
- **10 requests per minute** for call-triggering endpoints (start call, outbound)
- Rate-limited responses return HTTP `429 Too Many Requests`

## Common HTTP status codes

| Code | Meaning |
|------|---------|
| 200 | Success |
| 201 | Created |
| 400 | Bad request (validation error) |
| 401 | Unauthorized (invalid or missing API key) |
| 404 | Resource not found |
| 429 | Rate limited |
| 500 | Internal server error |
