---
title: "Cancellation, Deletion & Data Export"
---

If you need to close a workspace, shut down your full account, or export your data, the process is straightforward. The main rule is simple: export anything you need to keep before closure is processed.

### Closing a workspace

A single workspace can be closed without affecting the rest of your account.

This is especially useful for agencies or multi-client setups where one client environment needs to be shut down while others remain active.

When a workspace is closed:

- all agents in that workspace stop operating immediately,
- any phone numbers assigned to that workspace are released,
- and the workspace data becomes inaccessible through the platform.

This includes items such as:

- call history,
- recordings,
- transcripts,
- extracted data,
- and agent configuration data.

If you need to retain any of that information, export it before the workspace is closed.

### Closing your account

To close your full account, contact the RapidCall support team.

Account closure is handled through support rather than self-serve so that active workspaces, running campaigns, phone numbers, and final billing can be reviewed properly before access is removed.

When you request account closure, the support team will:

- confirm the request,
- process any final outstanding usage charges,
- release all phone numbers linked to the account,
- and deactivate account access.

If you need to retain any data, make sure it is exported before the closure is completed.

### Data export

You retain ownership of your data and can export it before closure.

Available export types include:

| Data Type | Export Format |
| :-- | :-- |
| Call history, metadata, outcomes, timestamps | CSV, XLS, JSON |
| Call recordings | ZIP archive of audio files |
| Transcripts | CSV, JSON |
| Agent configurations, prompts, settings | JSON |
| Contact or campaign data | CSV, XLS |
| Post-call extracted fields | CSV, JSON |

Call history exports can be generated directly from the **Call History** page inside the platform.

For larger or bulk exports covering recordings, transcripts, and agent configurations, contact support and the team will prepare the export for you.

### Important note

Once a workspace or account has been closed, the associated data is no longer accessible through the platform.

For that reason, it is strongly recommended that you complete all required exports before requesting closure.

### GDPR note

If you operate in the EU or UK, you may have legal obligations to delete personal data when requested by your contacts.

Call recordings, transcripts, and extracted data may contain personal data, which means they fall under your responsibilities as the data controller.

RapidCall allows you to:

- delete individual call records from **Call History**,
- and disable recording entirely at the agent level.

For broader guidance on compliance, see **Section 9: Compliance and Security**.
